# Seedance 2.0 Skill OS 中文版完整使用说明

> 适用版本：v6.6.0  
> 适合人群：不会写提示词、不会编程，但想用 Seedance 制作短视频、广告、剧情片、MV 或连续短剧的创作者  
> 推荐搭配：Codex + 即梦/Dreamina、豆包、剪映、火山方舟或其他已经接入 Seedance 2.0 的生成平台

## 一、先弄清楚：这套系统到底是什么

Seedance 2.0 Skill OS 不是视频生成网站，也不会自己替代即梦、豆包或火山方舟。

它是一套安装在 AI 助手里的“视频导演工作系统”。你负责说出想法、提供素材和决定是否采用，Codex 负责：

- 把模糊想法整理成能拍的方案；
- 判断应该用文生视频、图生视频、参考视频、首尾帧还是续拍；
- 为每张图片、视频和音频分配明确用途；
- 生成可以直接粘贴到 Seedance 的中文提示词；
- 把长故事拆成多个可生成的小片段；
- 记录人物、服装、道具、场景、动作和镜头连续性；
- 审查生成结果，判断保留、后期修复、重抽、局部编辑还是重写；
- 根据上一段视频的真实结尾继续写下一段；
- 在交付前检查画面、声音、字幕、版权和版本。

最简单的理解是：

```text
你提出创意并做决定
        ↓
Codex 做导演、拆镜头、写提示词、管理连续性
        ↓
你在 Seedance 平台上传素材并生成视频
        ↓
把生成结果交回 Codex 审片
        ↓
Codex 决定保留、修复、重做或继续下一段
        ↓
剪辑、字幕、音乐、调色和最终交付
```

## 二、正式使用前的准备

### 1. 安装到 Codex

在 Codex 中打开本项目目录 `E:\seedance-2.0`，运行项目自带的安装程序：

```powershell
python scripts\install_codex_skill.py --force
```

安装完成后，关闭并重新打开 Codex。新建任务时输入：

```text
$seedance-20
```

如果能够识别这个名称，说明安装成功。

安装只需做一次。以后制作新视频，不需要重新安装。

### 2. 准备一个实际的视频生成平台

这套系统负责“导演和提示词”，真正生成视频仍需在支持 Seedance 的平台完成，例如即梦/Dreamina、豆包、火山方舟或其他服务商。

不同平台支持的时长、清晰度、参考素材数量、标签写法、编辑和续拍功能可能不同。不要把一个平台的参数直接套到另一个平台。开始项目时告诉 Codex你使用的平台；如果不确定，Codex应采用保守方案，不编造参数。

### 3. 建议准备的素材

不一定每次全部需要，按项目选择：

- 角色参考图：锁定人物或原创角色身份；
- 产品参考图：锁定产品外形、颜色、标签和包装；
- 场景参考图：说明地点、环境或布景；
- 首帧图：规定视频从什么画面开始；
- 尾帧图：规定视频最终要到达什么画面；
- 动作参考视频：只参考动作、表演、运镜或节奏；
- 音频：只参考节奏、氛围、对白或环境声；
- 已经生成并确认采用的上一段视频：用于连续剧情或续拍；
- 上一段视频的最终一帧：用于观察真实结尾和稳定续拍。

只使用自己拥有、已授权、可商用或公共领域的素材。上传明星、真人脸、他人声音、影视角色、品牌 Logo、歌曲或影视片段，不代表你已经获得使用权。

## 三、每次开始时，先判断是哪一种任务

| 你的情况 | 系统应采用的路线 | 最终得到什么 |
|---|---|---|
| 只有一个模糊想法 | 创意访谈 | 明确的视频方案和第一版提示词 |
| 已经想清楚一个短场景 | 单条视频 | 一条可直接生成的提示词 |
| 一个故事要分几段拍 | 连续剧情 | 全局故事规划 + 只生成当前片段的提示词 |
| 要接着上一段继续 | 续拍 | 基于真实结尾的下一段提示词 |
| 画面基本满意但有小问题 | 审片与返工 | 保留、后期修复、局部编辑或重做建议 |
| 结果严重跑偏 | 故障诊断 | 原因、修复方案和重试提示词 |
| 涉及明星、影视角色、品牌、真人或歌曲 | 版权与安全改写 | 保留创意功能的原创或已授权方案 |

最重要的分界只有一个：

- 一次生成能讲清楚的一个动作或一个画面任务，按“单条视频”做；
- 需要多个动作、多个场景、连续剧情、人物对话或完整广告结构，按“连续剧情”做。

不要强行把完整故事塞进一条提示词。内容越多，人物、动作、镜头和画面越容易互相争抢。

## 四、第一次对 Codex 应该怎么说

### A. 只有一个模糊想法

复制下面这段，把方括号内容换成你的情况：

```text
请使用 $seedance-20 帮我把这个想法做成视频。

想法：[例如：猫掌柜深夜为顾客抢救一双有纪念意义的旧鞋]
发布平台：[抖音/小红书/朋友圈/广告投放]
视频生成平台：[即梦/豆包/火山方舟/暂时不确定]
画面比例：[9:16/16:9/1:1/暂时不确定]
目标时长：[例如 15 秒/30 秒/暂时不确定]
现有素材：[角色图、产品图、场景图、视频、音频，或没有]
是否需要对白：[需要/不需要/暂时不确定]

请先判断这是单条视频还是连续剧情。信息够用就直接推进；缺少关键信息时最多给我 3 个明确选项。
```

### B. 想法已经很清楚，只要一条视频

```text
请使用 $seedance-20 为下面场景生成一条可直接使用的中文提示词。

平台：[即梦/豆包/火山方舟/其他]
模式：[文生视频/图生视频/参考视频/首尾帧/不确定]
比例：[9:16]
时长：[以当前平台可用时长为准]
主体：[谁或什么产品]
本条只发生的动作：[一个清楚动作]
结束画面：[动作结束时必须看到什么]
想让观众感受到：[一个情绪或信息]
参考素材：[逐个说明，或让你替我分配用途]
声音：[对白/环境声/音效/音乐/静音]
不能改变：[人物、服装、产品、Logo、场景等]

请输出：素材上传顺序、每个素材的用途、最终提示词、生成参数注意事项、验收标准。
```

### C. 要做一段完整剧情、广告或 MV

```text
请使用 $seedance-20 按连续剧情项目帮我制作，不要一次把全部剧情塞进一条提示词。

项目名称：[名称]
最终成片用途：[短剧/MV/广告/品牌片]
故事梗概：[完整故事]
最终结局：[故事最后必须发生什么]
目标总时长：[例如 30 秒/1 分钟]
平台和比例：[即梦，9:16]
主角与固定形象：[说明或上传参考图]
现有素材：[列出]
声音方案：[对白/旁白/环境声/音乐]
不能改变的内容：[人物身份、服装、产品、世界观等]

请先完成故事主线、场景划分、连续性设定和片段地图；只把 Clip 01 写成最终提示词，后续片段先保留为计划卡。Clip 01 生成后，我会把视频交回来审片，再继续 Clip 02。
```

### D. 要接着上一段视频继续

先上传“已经确认采用的上一段视频”，最好同时上传它的最终一帧，然后说：

```text
请使用 $seedance-20 接着这段已经采用的视频继续制作。

项目名称：[名称]
上一段编号：[例如 Clip 01]
这段是否还在同一场景：[是/否/不确定]
下一段只需要完成：[一个任务]
后面暂时不能出现：[留给后续片段的内容]
必须保持：[人物、服装、道具、场景、光线方向等]
想让观众感受到：[一个感受]

请先观察上一段的真实结尾，不要按原计划猜测；再给我下一段的素材引用关系、连续性锁定和最终提示词。
```

### E. 生成结果不好，需要诊断

上传结果视频，并尽量附上使用过的提示词：

```text
请使用 $seedance-20 审查这个生成结果。

原提示词：[粘贴]
我最在意的问题：[脸变了/产品变形/动作不对/运镜乱/对白不同步/画面普通/接不上上一段]
哪些部分我想保留：[说明]

请判断应该保留、后期修、局部编辑、重新抽取还是重写提示词。先找根因，每次只改一个主要变量，不要简单堆更多形容词。
```

## 五、七种常用生成方式

| 方式 | 大白话解释 | 适合场景 | 关键要求 |
|---|---|---|---|
| 文生视频 T2V | 只靠文字生成 | 没有固定人物或产品的原创画面 | 主体、动作、镜头、光线、声音要明确 |
| 图生视频 I2V | 让一张图动起来 | 固定人物、产品、场景 | 少描述图片已有内容，只写新增动作和必须保持的内容 |
| 视频生视频 V2V | 以原视频为基础改变某一层 | 改光线、风格、背景或局部效果 | 明确“保留什么，只改什么” |
| 参考生成 R2V | 图片、视频、音频分别承担不同任务 | 锁身份 + 借动作 + 借节奏 | 每个素材只能有清楚的主要职责 |
| 首尾帧 FLF2V | 从指定起点自然走到指定终点 | 转场、变形、确定动作终点 | 首帧和尾帧都要明确，描述中间的连续过程 |
| 视频编辑 Edit | 对基本满意的视频只改失败层 | 画面和节奏对，只有光线或局部特效错 | 不要重新生成整个概念 |
| 延长/续拍 Extend | 从上一段真实结尾继续 | 同一镜头内的连续动作 | 必须以已采用的视频或尾帧为依据 |

平台是否支持某一种方式，要以当前平台实际界面为准。

## 六、参考素材怎么上传和引用

### 1. 给每个素材只分配一个主要职责

正确示例：

```text
@Image1 锁定猫掌柜角色身份和服装。
@Image2 只参考洗鞋工作台环境，不改变人物。
@Video1 只参考横向跟拍节奏，不复制视频里的人物、服装、地点和 Logo。
@Audio1 只控制节奏，画面动作在重拍点完成。
```

错误示例：

```text
参考所有素材的风格、人物、动作和氛围。
```

错误原因是模型无法判断谁负责人物、谁负责动作、谁负责场景，最后容易全部混在一起。

### 2. 标签必须一字不改

平台生成的引用标签，例如：

```text
@Image1  @Video1  @Audio1
@图片1   @视频1   @音频1
```

必须使用当前界面实际显示的标签，不能翻译、改编号、加空格或改成 `[图片1]`。上传顺序改变后，也要重新核对编号。

### 3. 写清楚“可以借什么”和“不能借什么”

```text
@Video1 只参考人物侧身行走的动作节奏；不复制人物长相、服装、房间、品牌和声音。
```

这句话比“参考这个视频”稳定得多，也能减少版权和身份误用风险。

### 4. 图生视频不要重复描述图片里已经存在的东西

图片已经告诉模型人物长相、衣服、构图、颜色和背景。提示词应重点写图片里看不到的内容：

- 接下来发生什么动作；
- 动作需要多少时间；
- 镜头怎样移动；
- 光线怎样变化；
- 有什么声音；
- 哪些东西绝对不能改变。

推荐结构：

```text
@Image1 为角色参考，严格保持脸部、发型、服装和场景布局不变；仅加入一次眨眼、轻微呼吸和缓慢抬头。镜头固定中近景，左侧窗光不变。无配乐，仅低环境声。不要新增人物、字幕和水印。
```

## 七、一条好提示词应该包含什么

一条实用提示词通常包含以下内容：

1. **参考关系**：哪个素材负责什么；
2. **主体**：谁是画面中心；
3. **一个可见动作**：动作要能看见结果；
4. **动作终点**：最后停在哪里、变成什么状态；
5. **一个主要镜头运动**：固定、推近、后拉、横移或跟拍；
6. **真实光源**：窗光、店内暖灯、路灯、月光或产品条形光；
7. **声音意图**：对白、环境声、音效、音乐或明确静音；
8. **保持和排除项**：哪些内容不能变，哪些内容不能提前出现。

不要只写：

```text
电影感、高级感、大片感、唯美、震撼、4K、顶级品质。
```

这些词没有告诉模型具体该做什么。把它们改成能拍出来的内容：

```text
固定中近景，猫掌柜擦去鞋面最后一层泡沫，鞋面露出干净反光后停手；左侧暖色工作灯照亮手套和鞋面，背景保持低照度；无配乐，只保留刷毛摩擦声和一声轻微呼气。
```

## 八、单条视频的完整操作流程

### 第 1 步：说清楚本条视频唯一要完成的任务

例如：

```text
让观众看到一双严重脏污的鞋，在专业清洗后露出干净鞋面。
```

不要同时要求进店、讲故事、清洗、对比、顾客感动、价格优惠和下单引导。

### 第 2 步：让 Codex 选择生成方式

有固定角色或产品，通常优先图生视频；有明确起点和终点，考虑首尾帧；要借动作或运镜，考虑参考视频；没有固定资产的原创画面，可以文生视频。

### 第 3 步：确认素材上传顺序

让 Codex 输出一张表，明确：文件名、上传顺序、平台标签、主要职责、禁止传递内容。

### 第 4 步：复制最终提示词到 Seedance

上传素材后，用平台实际生成的 `@` 标签替换或确认提示词中的标签，再提交生成。

### 第 5 步：先做便宜的小测试

如果平台允许，先用较短时长、草稿档或较低成本设置确认：构图、人物、动作方向和镜头是否正确。设计锁定后，再生成高质量完整版本。

### 第 6 步：把结果交回 Codex 审片

不要只说“效果不好”。上传视频，并说明最在意的一个问题。Codex 应给出明确判断：

- 保留；
- 后期修复；
- 局部编辑，不整条重做；
- 原提示词不变，换一次随机结果；
- 重写提示词；
- 换一种生成方式或拆成两条。

## 九、连续剧情的完整操作流程

连续剧情必须遵守一句话：**全局先规划，每次只生成当前一段。**

### 第 1 步：先定故事最终要到哪里

在 Clip 01 之前先确认：

- 故事目标；
- 最终结局；
- 按顺序发生的主要事件；
- 角色、服装、道具和场景固定规则；
- 整体导演风格；
- 画面、镜头、光线和声音怎样从开头走向高潮和结尾。

### 第 2 步：先分场景，再分片段

一个场景通常指相同地点和相近时间。场景内可以接着上一段续拍；换地点、换时间或进入新段落时，通常应主动切镜头，并重新使用原始角色参考图锁定身份。

长故事不应该一直无限续拍。同一条生成结果连续接两次左右后，就应考虑在合适的场景边界重新用原始角色图开镜头，防止人物、服装和场景越接越偏。

### 第 3 步：每个片段只承担一个叙事任务

例如一个三段故事：

| 片段 | 本段只做什么 | 本段结束状态 | 不能提前出现 |
|---|---|---|---|
| Clip 01 | 猫掌柜收到一双严重脏污的旧鞋 | 鞋被放到检查台上 | 不开始清洗，不展示成品 |
| Clip 02 | 检查并开始处理主要污渍 | 污渍开始松动 | 不展示完全洗净，不出现顾客 |
| Clip 03 | 展示清洗完成并交还顾客 | 顾客接过干净鞋 | 不重复收鞋和清洗动作 |

### 第 4 步：只定稿 Clip 01

Codex 可以规划后面的片段，但不能提前把 Clip 02、Clip 03 写成最终提示词。因为 Clip 01 实际生成的结尾可能和计划不同。

### 第 5 步：生成 Clip 01 并回传

选择你愿意采用的一条结果，把完整视频交回 Codex。没有完整视频时，至少提供最终一帧。

如果需要在本地提取最终一帧，可以在项目目录运行：

```powershell
python scripts\extract_last_frame.py "你的视频文件.mp4"
```

### 第 6 步：以真实结果更新项目事实

如果计划要求人物走到门边，但视频实际停在门前两步，只要你决定采用这条视频，下一段就必须从“门前两步”开始，而不能假装人物已经到门边。

系统应记录：

- 人物最后的位置、姿势和视线；
- 仍在继续的动作方向；
- 镜头最后处于什么运动阶段；
- 服装、道具归属和道具状态；
- 场景、光线、环境声；
- 已经完成的剧情；
- 后面才能发生的剧情。

### 第 7 步：再定稿下一段

下一段提示词必须：

- 从上一段真实结尾开始；
- 不重复已经发生的动作；
- 不提前泄露更后面的情节；
- 保持人物、服装、道具、方向、光线和声音连续；
- 只完成当前片段的一个任务。

### 第 8 步：场景边界主动重置

换地点、换时间或连续续拍开始漂移时，使用原始角色图、产品图和场景参考重新开一个镜头。这里追求的是故事连续，不必冒险承诺画面无缝连接。

### 第 9 步：保存项目状态卡

每次结束工作前，让 Codex 输出“项目状态卡”，至少包含：

- 项目名称和当前片段编号；
- 已完成场景；
- 当前真实结尾；
- 已完成动作；
- 下一段只做什么；
- 人物、服装、道具和场景锁定项；
- 后续保留内容；
- 当前使用的参考素材标签；
- 尚未确认的问题。

下次新开 Codex 任务时，把状态卡和上一段已采用的视频一起提供，就能继续，而不依赖聊天记忆。

## 十、生成结果回来后，怎么决定是否重做

每条结果只做五种判断：

| 判断 | 什么时候用 | 下一步 |
|---|---|---|
| 保留 | 核心画面任务已经完成，没有致命问题 | 锁定版本，继续下一条 |
| 后期修复 | 颜色、字幕、音量、剪掉头尾几帧等问题 | 交给剪辑，不浪费生成次数 |
| 局部编辑 | 构图和动作正确，只有一个画面层错误 | 保留源视频，只改错误层 |
| 重新抽取 | 提示词本身正确，只是这次随机结果不好 | 原提示词不变，再试 1 至 2 次 |
| 重写 | 相同问题连续出现两次 | 停止碰运气，找原因并改提示词 |

### 一次只改一个变量

每次返工只改一项：

- 一句话；或
- 一个参考素材；或
- 一个随机种子；或
- 一种生成方式。

不要同时改动作、镜头、素材、时长和风格，否则即使结果变好，也不知道是哪一项起作用。

### 生成前设置尝试上限

建议一开始就约定最多尝试多少次，以及什么叫“可以采用”。达到一半次数仍在重复同一个错误时，应换方案，例如缩短片段、减少人物、简化手部动作、改用首尾帧或拆成两个镜头。

## 十一、常见问题和处理方法

| 问题 | 常见原因 | 最先怎么改 |
|---|---|---|
| 人脸或产品变了 | 重复描述了图片已有内容，或动作太多 | 减少静态描述，强化身份/产品保持项 |
| 镜头乱跳 | 同时要求推、拉、摇、绕拍 | 只保留一个镜头运动并写清终点 |
| 动作没发生 | 只写了静态形容词，没有动作结果 | 写清谁做什么、何时完成、最后变成什么 |
| 画面很普通 | 只写电影感、高级感 | 改成真实动作、光源、材质和声音 |
| 手部变形 | 手部动作太细、接触关系复杂 | 简化手部动作，换角度或把细节拆开拍 |
| 对白口型不稳 | 台词太长、人物转头、镜头运动太多 | 固定中近景、缩短台词、对白时不转头 |
| 特效很乱 | 没写特效从哪里来、怎样运动、怎样结束 | 写清来源、材质、路径、互动和消散终点 |
| 文字或 Logo 破损 | 要求模型在运动中重画小字 | 尽量保持静止，正式字幕和文案后期添加 |
| 接不上上一段 | 没看真实结尾，按计划猜测 | 上传上一段或尾帧，从实际状态继续 |
| 重复上一段动作 | 没记录已完成事件 | 明确“不要重演上一段动作” |
| 后面剧情提前出现 | 一条提示词塞得太多 | 写明“本段停止在……，后续……暂不出现” |
| 连续几段后人物漂移 | 一直用生成结果继续套生成结果 | 在场景边界用原始角色参考重新开镜头 |
| 音频参考没生效 | 视频原声和音频在争夺节奏 | 静音动作参考视频，明确音频只控制节奏 |

## 十二、对白、字幕、音乐和声音

### 对白

- 一次只给一个角色一句短台词；
- 写清角色名、动作和台词；
- 对白期间尽量固定中近景；
- 不要同时大幅转头、快速走动和复杂运镜；
- 平台有“唇形同步”开关时，要确认已经开启；
- 口型属于容易失败的环节，要预留重试次数。

示例：

```text
猫掌柜保持正面中近景，轻轻放下鞋刷，说：“放心，这双鞋还能救回来。”对白期间镜头固定，不转头。无配乐，只保留安静店内环境声。
```

### 字幕和广告文案

正式字幕、价格、优惠信息、品牌口号、法务文案和复杂 Logo，建议在剪映、Premiere、达芬奇等剪辑软件中添加，不要把最终文字交给视频模型生成。

### 音乐

多段视频分别生成时，音乐通常不能天然无缝衔接。每段可以生成环境声、动作音效和短对白，整条片子的统一音乐、节拍和混音放到后期完成。

## 十三、交付前检查清单

### 画面

- 比例、分辨率和画面裁切正确；
- 人脸、服装、产品形状、Logo 和颜色稳定；
- 没有闪烁、严重变形、穿帮和多余人物；
- 竖屏裁切时主体和产品仍在安全区域；
- 所有脆弱画面都逐帧快速检查过。

### 声音

- 对白与口型基本同步；
- 环境声、音效和音乐音量合理；
- 不同片段之间没有突兀的音量跳变；
- 需要配音或多语言版本时保留无对白/无音乐素材。

### 字幕与文字

- 字幕没有错字；
- 文字处在安全区域；
- 价格、活动信息和免责声明准确；
- 有一份无字幕、无文案的干净版本。

### 连续性

- 人物服装、发型和道具一致；
- 动作方向和视线方向没有突然反转；
- 产品朝向和场景布局一致；
- 已完成动作没有重复；
- 前后片段的光线方向和时间关系说得通。

### 版权和授权

- 角色、音乐、声音、图片、视频、Logo 和真人肖像都有明确使用依据；
- 没有用换语言、改拼写等方式绕过内容审核；
- 保存素材来源、生成平台、提示词版本和审核负责人记录。

## 十四、文件命名和项目整理建议

建议为每个视频建立独立文件夹：

```text
项目名称/
├─ 01_方案与脚本/
├─ 02_角色与产品参考/
├─ 03_场景参考/
├─ 04_提示词/
├─ 05_生成原片/
├─ 06_已采用片段/
├─ 07_项目状态卡/
├─ 08_剪辑工程/
└─ 09_最终交付/
```

片段文件建议这样命名：

```text
项目名_SC01_CLIP01_TAKE01_待审.mp4
项目名_SC01_CLIP01_TAKE02_采用.mp4
项目名_SC01_CLIP02_TAKE01_退回.mp4
```

最终文件建议包含：项目、版本、比例、语言、日期和状态，例如：

```text
猫掌柜_洗鞋故事_HERO_V03_9x16_ZH_2026-07-13_FINAL.mp4
```

## 十五、推荐的日常协作方式

为了减少浪费，建议固定采用下面的循环：

1. 你给创意、素材和目标；
2. Codex 判断单条还是连续剧情；
3. Codex 给出素材上传表、当前提示词和验收标准；
4. 你在 Seedance 生成；
5. 你把结果原片交回 Codex；
6. Codex 给出五类审片结论之一；
7. 采用后更新项目状态卡；
8. 只在真实结果基础上继续下一段；
9. 所有片段完成后统一剪辑、字幕、音乐和质检。

不要连续生成很多条后才回来检查。最稳妥的节奏是：**一段生成，一段审片，一段确认，再进入下一段。**

## 十六、你真正需要记住的十条规则

1. 这套系统负责导演和提示词，Seedance 平台负责生成视频。
2. 一个片段只完成一个主要任务。
3. 长故事先做全局规划，但只定稿当前片段。
4. 每个参考素材只承担一个主要职责。
5. `@Image1`、`@Video1`、`@Audio1` 等标签必须原样保留。
6. 图生视频只写图片没有告诉模型的内容。
7. 一条提示词只安排一个主要运镜。
8. 上一段真实结果比原计划更有优先级。
9. 生成失败先找原因，每次只改一个变量。
10. 字幕、价格、复杂 Logo、统一音乐和最终调色放到后期完成。

## 十七、第一次实操建议

第一次不要直接做一分钟短剧。建议用一个 5 至 10 秒的简单图生视频完成闭环：

1. 准备一张主体清楚、构图稳定的图片；
2. 让 Codex 只设计一个动作；
3. 上传图片并确认引用标签；
4. 生成一条短视频；
5. 把结果交回 Codex 审片；
6. 根据结论只改一个变量再试一次；
7. 掌握流程后再做三段连续剧情。

可直接使用的第一条练习指令：

```text
请使用 $seedance-20 带我完成第一次 Seedance 图生视频练习。我会提供一张图片。请你负责判断一个最稳妥的动作，输出素材引用、中文提示词、生成前检查和生成后验收标准。一次只推进一个步骤，我生成后会把视频交回来给你审片。
```
