# 猫掌柜 AI 漫剧创作工作台：全流程对话提示词模板

版本：1.0  
适用范围：剧情短片、创意广告、音乐 MV  
配套文档：`AGENTS.md`、`WORKFLOW.md`

这份文档给工作台用户直接使用。每个代码块都是可以复制到新对话或当前对话里的提示词。你不需要懂 Agent、MCP、数据库或代码，只需替换 `{花括号里的内容}`。

## 一、使用原则

1. 每次更换大模型或新开对话，先使用“模板 01”。
2. 新项目使用“模板 02”，已有项目使用“模板 03”。
3. AI 应自动推进内部流程，遇到用户审核、视觉风格确认或真实生成时停下来。
4. 图片、视频、音频真实生成只能由你在工作台点击按钮触发。对话中的 AI 不得代替点击或擅自产生费用。
5. 工作台阶段和最新版本是唯一依据。聊天里说“已经完成”，但工作台没有记录时，视为没有完成。
6. 不确定当前该用哪条模板时，直接使用“模板 04：读取状态并继续”。

## 二、开始前准备的信息

新项目最好准备这些内容；没有的可以留空，让 AI 给出明确选项：

```text
项目名称：
内容模式：剧情短片 / 创意广告 / 音乐MV / 不确定
一句话创意：
目标观众：
发布平台：默认抖音
画幅：默认9:16
目标时长：
希望观众产生的情绪或行动：
已经确定的视觉风格：
不能出现的内容：
已有角色、产品、图片、音乐或参考素材：
预算偏好：先低成本样片 / 质量优先 / 不确定
```

## 三、新开对话与接手工作台

### 模板 01：让新 AI 熟悉工作台

用途：每次新开对话、更换模型或更换 AI 工具时，第一条发送。

```text
你现在接手“猫掌柜 AI 漫剧创作工作台”。先不要创作、不要改项目、不要提交任何图片/视频/音频生成任务。

请先完整阅读项目根目录的 AGENTS.md、WORKFLOW.md，以及 WORKFLOW 要求的项目级 Skill。然后按以下顺序做只读检查：

1. 检查 creative-production-orchestration、gpt-image-2-storyboard、doubao-audio-generation、seedance-20 四个 Skill 的验证状态。
2. 读取工作台项目列表，但不要创建新项目。
3. 检查当前可用的专业 Agent；如果运行环境没有子 Agent，说明你将如何按 WORKFLOW 的“隔离角色轮次”执行。
4. 检查工作台健康状态、当前流程门禁和真实生成权限。
5. 用大白话向我报告：工作台是否正常、有哪些项目、是否存在待审核/待处理/失败任务、下一步有哪些明确选项。

必须遵守：
- 不凭聊天记忆猜工作台状态。
- 不直接修改数据库。
- 不把抖音自动理解为港风。
- 视觉风格未锁定时不进行真实生成。
- 没有我在工作台点击生成按钮，不得调用任何会产生费用或消耗额度的真实生成接口。
- 创建、审核和用户确认必须分开，不能自写自批。
```

### 模板 02：创建一个新项目

```text
请按照猫掌柜工作台 WORKFLOW 创建一个新项目，并使用 creative-production-orchestration 组织流程。

项目名称：{项目名称}
一句话创意：{一句话创意}
内容模式：{剧情短片 / 创意广告 / 音乐MV / 请你判断}
目标观众：{目标观众}
发布平台：{默认抖音}
画幅：{默认9:16}
目标时长：{例如60秒}
情绪目标：{例如先好奇、再紧张、最后温暖}
传播或转化目的：{例如完播和评论 / 到店成交 / 品牌记忆}
视觉风格：{已经明确的风格；不明确就写“待确认”}
已有素材：{没有就写“无”}
禁止内容：{没有就写“无”}
预算偏好：{先做低成本代表样片 / 质量优先}

请先由 creative-producer 建立创作档案。信息足够时直接推进；只有缺失信息会实质改变项目方向时，才向我提出1—3个明确选项，不要问开放式问题。

创作档案确认后可以自动推进到“剧本等待我审核”，但不能越过以下门禁：
- 视觉风格需要我确认时必须停下。
- 导演审核与观众审核必须分别完成并绑定同一最新剧本版本。
- 未经我在工作台确认，不得进入资产真实生成。
- 不得自行提交任何真实生成任务。
```

### 模板 03：接手一个已有项目

```text
请接手工作台中的项目“{项目名称或项目ID}”。

先读取完整项目上下文、当前阶段、最新版本、视觉风格状态、未解决退回项、资产、分镜和生成任务。不要沿用聊天中的旧结论。

请用以下格式报告：
1. 当前阶段。
2. 最新锁定产物和版本。
3. 已通过与待通过的门禁。
4. 未解决退回项。
5. 正在等待、失败或已完成的生成任务。
6. 当前唯一应该做的下一步。

报告后，如果下一步不涉及用户审核、风格确认或真实生成费用，就按 WORKFLOW 自动推进；否则停下来告诉我应该在工作台做什么。
```

### 模板 04：不知道当前进行到哪里时

```text
请读取工作台当前项目状态，不要猜测。按照 WORKFLOW 判断现在处于哪个阶段、卡在哪个门禁、当前唯一允许的下一步是什么。

如果可以安全自动推进，就调度对应专业 Agent 完成并写回工作台；如果需要我审核、确认视觉风格或点击真实生成按钮，请停止并用大白话告诉我具体要点击哪里、审核什么、通过后会进入哪个阶段。
```

## 四、创作档案与剧本阶段

### 模板 05：补全或修改创作档案

```text
请读取项目“{项目名称或ID}”的最新创作档案，并由 creative-producer 检查以下内容是否可执行：内容模式、目标观众、平台、画幅、时长、情绪曲线、传播/转化目的、预算和验收标准。

我的补充或修改是：{具体内容}

请只修改受影响字段，不要重写已经确认的内容。视觉风格必须写出可执行的风格名称、光线、色彩、镜头、材质和时代依据；证据不足就标记 needs_review，不得自动补成港风、胶片、霓虹或90年代香港电影。
```

### 模板 06：确认视觉风格

```text
请为项目“{项目名称或ID}”整理视觉风格确认单，不要生图。

我希望的方向：{例如现代现实主义、自然窗光、克制暖色、干净生活质感}
我不希望出现：{例如港风霓虹、过度胶片颗粒、赛博朋克}
可参考的已有素材：{素材说明或“无”}

请输出并写入：
- 风格名称
- 可见风格描述词
- 光线来源
- 主色和辅助色
- 镜头与构图倾向
- 材质与时代依据
- 风格证据来源
- 禁止自动添加的风格

如果这些信息足够，请将视觉风格锁定；如果仍不足，只给我最多3个明确选项，不要自行猜测。
```

### 模板 07：编写第一版剧本

```text
请读取项目“{项目名称或ID}”最新创作档案，由 screenwriter 编写可拍、可视化、可拆分镜头的第一版剧本。

额外要求：{没有就写“按创作档案执行”}

每场戏必须写清：
- 场景和时间
- 出场角色
- 人物目标
- 阻力或冲突
- 可见动作
- 对白与声音线索
- 本场发生的变化
- 结果以及如何推动下一场

不要在剧本中编写最终生图或视频模型提示词，不要擅自增加未确认视觉风格。完成后保存为新的剧本版本，并进入内部审核。
```

### 模板 08：让 AI 完成内部审核

```text
请处理项目“{项目名称或ID}”当前最新剧本的内部审核。

严格按顺序执行：
1. director-reviewer 审核戏剧核心、电影感、节奏和可拍性。
2. 只有当前剧本版本通过导演审核后，audience-reviewer 才审核前三秒钩子、理解成本、情绪推进和完播动力。
3. 两项审核必须绑定同一个最新剧本ID；不得复用旧剧本的通过记录。
4. 通过标准为平均分不低于4，任何关键项不低于3。
5. 退回时把问题写到具体责任对象并创建新版本；最多自动返工3轮。

两项都通过后，把项目推进到“等待我审核剧本”，并告诉我这版剧本最值得重点检查的3件事。不要替我完成用户审核。
```

### 模板 09：我退回了剧本

```text
我已经在工作台退回项目“{项目名称或ID}”的剧本。请先读取最新未解决退回项，不要让我重复粘贴意见。

由 screenwriter 只修改退回意见影响的部分，保留已通过内容，创建新剧本版本。然后重新走 director-reviewer 和 audience-reviewer 两项独立审核。

完成后告诉我：改了什么、为什么这样改、哪些内容保持不变、当前是否已经重新进入用户剧本审核。
```

### 模板 10：我已通过剧本

```text
我已经在工作台通过项目“{项目名称或ID}”的最新剧本。请读取工作台确认状态已经进入 asset_design，再按照 WORKFLOW 开始资产设计。

不要覆盖锁定剧本，不要自行提交真实生图。先由 asset-designer 建立完整资产清单，再由 image-prompt-designer 使用 gpt-image-2-storyboard 编写和检查最终资产图提示词。
```

## 五、资产设计与资产图阶段

### 模板 11：建立完整资产清单

```text
请为项目“{项目名称或ID}”读取最新锁定剧本和视觉风格，由 asset-designer 建立完整资产方案并写入工作台。

资产至少分为：
- 角色及必要的服装/状态变化
- 场景及时间/天气状态
- 关键道具
- 整体风格资产

每个资产写清：名称、类型、结构化引用编号、可见描述、固定身份锚点、在故事中的用途、连续性要求、参考图职责和禁止变化项。

随后把每个最终资产图提示词交给 image-prompt-designer，强制使用 gpt-image-2-storyboard Skill 检查。资产ID和项目名只能保存在结构化字段里，不得写入发给生图模型的自然语言提示词。

角色设定图必须是3:2横向参考拼版：左侧约40%脸部特写，右侧约60%正面/侧面/背面全身三视图；浅灰中性背景、柔和均匀棚光，不得写成剧情电影静帧。

只准备资产和提示词，不要提交真实生图。
```

### 模板 12：检查现有资产提示词

```text
请检查项目“{项目名称或ID}”当前全部资产提示词。使用 gpt-image-2-storyboard Skill 和对应 prompt lint，逐项报告通过或不通过。

重点检查：
- 是否把资产ID、项目名或版本号混入模型提示词
- 角色图是否符合3:2左脸右三视图硬规范
- 场景图是否交代完整空间并避免无关人物
- 道具图是否使用干净可比较的产品式构图
- 资产图是否错误使用了剧情霓虹、雨夜或戏剧光效
- 身份锚点、材质、颜色和禁止变化项是否可执行

只修复不合格提示词并保存新版本，不要生成图片。
```

### 模板 13：处理工作台待生图任务

用途：只有你已经在工作台点击“使用 Codex 额度”后才发送。

```text
处理工作台待生图任务。

请先读取 list_pending_codex_image_requests，只处理我已经在工作台创建的请求。逐项领取后，严格使用请求中的提示词、画幅、质量和参考图，通过内置图片生成能力完成，并把真实本地结果导回工作台。

要求：
- 不创建额外任务，不切换到APIMart，不擅自增加生成张数。
- 生成前再次检查角色资产图硬规范和参考图职责。
- 成功后写回真实图片路径；失败就写回真实错误原因。
- 每项任务都必须完成或明确标记失败，不能长期停在处理中。
- 全部处理后告诉我哪些资产已生成、哪些失败、下一步应该在资产中心审核什么。
```

### 模板 14：我退回了某个资产

```text
我已经在工作台退回项目“{项目名称或ID}”中的资产“{资产名称；不知道可留空}”。

请先读取最新退回项，由 generation-supervisor 判断主要原因属于：资产设计、身份锚点、提示词、参考图职责、模型参数还是生成偶发问题。

只修改受影响资产和引用它的镜头，不要重做其他已通过资产。保留正确部分，每次只改变一个主要变量。修改后的最终提示词必须重新通过 gpt-image-2-storyboard lint。

只准备新版本，不要自行提交真实生图。完成后告诉我应该在工作台重新生成哪个资产。
```

### 模板 15：资产全部通过后继续

```text
我已经在工作台逐项通过并锁定项目“{项目名称或ID}”的全部资产。请重新读取项目状态，确认是否还需要我点击“完成资产审核”。

如果整体资产审核仍待用户确认，请停下来告诉我点击位置，不得替我通过；如果工作台已经进入 storyboard_design，再按 WORKFLOW 调度 cinematographer、storyboard-artist、image-prompt-designer、video-prompt-designer 和 audio-supervisor 建立完整分镜，不要跳到真实生成。
```

## 六、分镜、首帧、视频和声音阶段

### 模板 16：生成完整分镜方案

```text
请为项目“{项目名称或ID}”读取最新锁定剧本、视觉风格和已通过资产，生成完整可生产分镜并写入工作台。

严格分工：
- cinematographer 先设计景别、构图、机位、镜头运动、焦点、光线、色彩和连续性。
- storyboard-artist 再拆成完整镜头表。
- image-prompt-designer 使用 gpt-image-2-storyboard 编写每个镜头的首帧提示词。
- video-prompt-designer 使用 seedance-20 编写每个镜头的视频提示词。
- audio-supervisor 使用 doubao-audio-generation 设计声线、对白、环境声、音乐和音效。

每个镜头必须包含：
- 镜头编号、标题、叙事目的、4—15秒时长
- 场景和引用资产
- 景别、构图、镜头、动作、对白和声音
- 画面开始时的静态首帧状态
- plannedStartState、plannedEndState
- sequenceRelation、parentShotId
- alreadyHappened、reservedForLater
- continuityLocks、allowedChanges
- imagePrompt、videoPrompt
- audioMode、声音资产和口型说明

分镜首帧提示词只描述画面开始时可见状态；视频提示词描述该镜头时长内真正发生的动作。连续镜头不得提前使用尚未通过视频的计划尾帧。

完成后保存新的 storyboard 版本并停在用户分镜审核，不要提交真实生成。
```

### 模板 17：检查分镜是否可以生产

```text
请对项目“{项目名称或ID}”当前最新分镜做一次生产前检查，不生成任何媒体。

检查：
1. 是否覆盖锁定剧本全部必要事件。
2. 每个镜头是否只有一个清楚的主要事件。
3. 单镜头是否在4—15秒内。
4. 角色、服装、道具和空间是否正确引用已锁定资产。
5. 首帧提示词和视频提示词是否混写。
6. Seedance连续性字段是否完整。
7. 对白时长、口型、音频模式和参考音频是否匹配。
8. 广告是否有品牌/CTA检查；MV是否有音乐同步和高潮检查。
9. 是否存在无法生成、成本异常或理解歧义。

发现问题时只修改相关镜头并创建新版本；全部合格后停在用户分镜审核。
```

### 模板 18：我退回了分镜

```text
我已经在工作台退回项目“{项目名称或ID}”的分镜。请先读取最新退回项，不要让我重复意见。

由 generation-supervisor 把意见分配给 cinematographer、storyboard-artist、image-prompt-designer、video-prompt-designer 或 audio-supervisor。只修改受影响镜头和字段，保留已正确内容，创建新的分镜版本并重新做生产检查。

不要提交任何真实生成。完成后列出修改的镜头编号、修改前问题、修改后结果和保持不变的内容。
```

### 模板 19：分镜通过，准备代表性首帧

```text
我已经在工作台通过项目“{项目名称或ID}”的最新分镜。请读取状态确认已经进入 sample_image。

请选择最能同时暴露角色一致性、场景、构图、光线和关键动作风险的一个代表性镜头，说明为什么选它。再次检查该镜头首帧提示词、资产引用和参考图职责。

只把代表性镜头准备到“可以由我在工作台点击生图”的状态，不要替我提交真实任务。告诉我应该在哪个镜头点击哪个生图通道。
```

### 模板 20：代表性首帧退回

```text
我已经退回项目“{项目名称或ID}”的代表性首帧。请读取工作台退回项和生成记录，由 generation-supervisor 判断问题属于资产、提示词、参考图、模型参数还是偶发生成。

只修改主要责任对象：
- 脸、服装、体型漂移优先修资产或身份锚点。
- 空间和道具关系错误优先修场景/分镜。
- 构图、光线或静态动作错误优先修首帧提示词。

保留正确部分，只改一个主要变量并重新通过图像 lint。不要自行重生图，告诉我应重新生成哪个镜头。
```

### 模板 21：代表性首帧通过，准备视频

```text
我已经在工作台锁定项目“{项目名称或ID}”的代表性首帧。请读取状态确认进入 sample_video。
```

### 模板 26：最终成片审核


## 九、推荐的完整使用顺序

一个新项目通常按下面顺序使用模板：

1. 模板 01：让新 AI 熟悉工作台。
2. 模板 02：创建项目。
3. 模板 06：必要时确认视觉风格。
4. 模板 07—08：写剧本并完成内部审核。
5. 你在工作台审核剧本；退回用模板 09，通过用模板 10。
6. 模板 11—12：建立并检查资产。
7. 你在工作台点击资产生图；Codex 通道用模板 13。
8. 资产退回用模板 14；全部通过后用模板 15。
9. 模板 16—17：建立并检查分镜。
10. 分镜退回用模板 18；通过后用模板 19。
11. 代表首帧退回用模板 20；通过后用模板 21。
12. 代表视频退回用模板 22；通过后用模板 23。
13. 批量阶段退回用模板 24。
14. 全部镜头通过后使用模板 25—26。
15. 成片退回用模板 27；最终通过后使用模板 28。

任何时候不确定，都回到模板 04，让 AI 读取工作台状态后判断，不要凭感觉跳阶段。

## 十、禁止使用的危险说法

不要只对 AI 说下面这些模糊指令：

- “全部自动做完。”
- “你看着生成就行。”
- “跳过审核直接出片。”
- “把所有图片和视频一次性生成。”
- “随便找个风格。”
- “把不好的全部重做。”
- “不用看工作台，按之前聊天继续。”

这些说法容易造成越级、重复费用、版本污染或资产漂移。正确做法是使用本文件对应阶段模板，让 AI 先读取真实状态，再推进当前唯一合法步骤。
